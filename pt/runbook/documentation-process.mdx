# 📝 Processo de Documentacao (PT-BR)

> Garantir consistencia EN/PT e qualidade tecnica.

## 🔁 Fluxo

1. Abrir PR com escopo claro
2. Review tecnico (precisao + comandos)
3. Review de linguagem (paridade EN/PT)
4. Merge somente apos aprovacoes

## ✅ Regras de consistencia

- Toda mudanca EN exige espelho PT no mesmo PR ou issue vinculada
- Todo comando de exemplo deve ser executavel
- Toda env var citada deve existir no `.env.example`

## 📚 Politica de changelog

Atualizar [Documentation Updates](../changelog/documentation-updates) a cada release relevante.
