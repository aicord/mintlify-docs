# 🛠️ Runbook de Operacao (PT-BR)

> Playbook diario para operacao e incidentes.

## 🔄 Ciclo do servico

- ▶️ Start: comando do process manager
- 🔁 Restart: rolling restart quando houver mais de uma instancia
- ⏹️ Stop: somente janela de manutencao

## 📅 Checks diarios

1. Status do health endpoint
2. Taxa de erro nos logs
3. Tendencia de p95
4. Pressao de fila
5. Taxa de fallback upstream

## 🗓️ Checks semanais

1. Auditoria de rotacao de chave
2. Janela de update de dependencias
3. Validacao de backups (config/env/docs)

## 🚨 Processo de incidente

1. Classificar severidade (P1/P2/P3)
2. Registrar sintomas e timeline
3. Mitigar primeiro (failover/limites temporarios)
4. RCA apos estabilizacao
5. Publicar resumo interno

## 📦 Politica de deploy

- PR obrigatoria para mudancas operacionais
- Sem alteracao direta em producao sem registro
- Plano de rollback antes de release
