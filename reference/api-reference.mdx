# 📡 API Reference (AiCordCloud)

> OpenAI-compatible gateway endpoints and response contract.

## 🌍 Base URL

```text
https://api.aicord.cloud
```

## 🔐 Authentication

Use one of:

- `Authorization: Bearer <API_KEY>`
- `x-api-key: <API_KEY>`

## 🧩 Endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/` | Service metadata |
| `GET` | `/health` | Health signal |
| `GET` | `/v1/models` | Public model aliases |
| `POST` | `/v1/chat/completions` | Chat completion |

### Health response

```json
{"ok": true}
```

### Chat completion example

```bash
curl https://api.aicord.cloud/v1/chat/completions \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "aicord/fast",
    "messages": [
      {"role":"system","content":"You are a coding assistant."},
      {"role":"user","content":"Create a Discord slash command /ping."}
    ],
    "temperature": 0.2
  }'
```

## 🏷️ Response headers

- `x-aicord-tier`
- `x-aicord-model`
- `x-aicord-upstream`
- `x-aicord-latency-ms`
- `x-ratelimit-remaining`

## ❌ Error format

```json
{
  "error": {
    "message": "Unauthorized",
    "type": "authentication_error"
  }
}
```

## 📄 OpenAPI spec

- Local path: `openapi/aicordcloud.openapi.yaml`
- Validate:

```bash
npx mintlify@latest openapi-check docs/openapi/aicordcloud.openapi.yaml
```
